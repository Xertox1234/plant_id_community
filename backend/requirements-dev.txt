# Development dependencies.
#
# This file is intentionally a thin overlay of requirements.txt — it carries NO
# version pins of its own. requirements.txt already includes every dev/test tool
# (pytest, pytest-django, mypy, django-stubs, bandit, safety, factory_boy, Faker,
# freezegun, coverage, ipython, django-debug-toolbar, …), so a separate frozen
# copy here only drifts.
#
# History (todo 217): this used to be a full pip-freeze that silently fell behind
# requirements.txt — it pinned Django 5.2.7 / wagtail 7.1.2 while prod, CI, and
# every local venv ran Django 6.0.x / wagtail 7.4. That dev/prod split papered
# over the Django 6.0 `format_html` admin 500 (which "worked on 5.x"), and forced
# recurring one-off bumps (PyJWT, wagtail-ai). Pointing at requirements.txt makes
# dev == prod by construction, so there is nothing left to diverge.
#
# Do NOT add pinned packages below. New dev-only tools go in requirements.txt
# (the single source of truth). If prod/dev ever genuinely need to differ, split
# a requirements-prod.txt base first rather than re-forking pins here.
